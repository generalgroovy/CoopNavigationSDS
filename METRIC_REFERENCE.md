# Metric Reference

Generated by `python scripts/generate_research_docs.py`.
Every listed metric is obligatory. A metric without sufficient evidence is retained with a null value and an explicit reason.

The runtime captures raw phase evidence first and computes metrics retrospectively. Each `metrics_long.csv` row stores the metric key, phase, numeric value, range, preferred direction, formula, operands, substituted calculation, and unavailable reason. This file mirrors the same catalog metadata used by the exporter.

## Calculation Algorithm Classes

| Class | Meaning | Implementation behavior |
| --- | --- | --- |
| D | Deterministic trace metric | Counts, ratios, edit distances, timing aggregations, route comparisons, and state comparisons over logged evidence. |
| R | Reference audio metric | Uses stored waveform evidence and an available reference or estimator; unavailable when the required model/reference is absent. |
| L | Learned estimator | Uses an optional learned evaluator such as NISQA or DNSMOS; unavailable when local assets or dependencies are missing. |

Common algorithm families are: ratio/count metrics use `eligible numerator / eligible denominator`; edit-distance metrics use deterministic substitutions, deletions, and insertions; timing metrics aggregate recorded phase durations; route metrics compare logged routes against precomputed staged optima; cross-run metrics use completed `MetricRecord` rows only.
