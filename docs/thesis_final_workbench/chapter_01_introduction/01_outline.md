# Chapter 1: Introduction - Outline

Use this as the direct writing plan for the final compact thesis structure.

## Function

- Motivate automatic evaluation of spoken task-oriented dialogue systems.
- State the research gap: final task success alone does not explain phase-level failure.
- Define research questions and contributions without implementation detail.

## Sections

### 1.1 Motivation

- spoken route assistance is practical
- speech introduces naturalness and accessibility
- task-oriented dialogue gives checkable goals

### 1.2 Evaluation Problem

- spoken SDS failure can originate in TTS, ASR, NLU, state, policy, grounding, or NLG
- automatic evaluation must preserve phase evidence

### 1.3 Research Questions

- phase-wise evidence
- metric-outcome relation
- failure localization
- backend comparison
- text versus speech

### 1.4 Contributions

- controlled route-dialogue framework
- retrospective evidence logging
- metric validity analysis
- matched text/audio comparison

## Keep Out

- exhaustive metric catalogs;
- raw per-run tables;
- long prompt templates;
- implementation details that do not support the chapter argument.
